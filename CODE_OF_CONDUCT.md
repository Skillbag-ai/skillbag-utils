# Code of Conduct

## Our Standard

Contributors should keep discussions technical, respectful, and constructive.

Expected behavior:

- focus on the skill behavior, repository text, or design tradeoff
- explain disagreements with evidence and concrete reasoning
- keep proposed changes scoped and reviewable
- help improve clarity instead of adding ambiguity

Unacceptable behavior:

- personal attacks
- harassment or intimidation
- bad-faith disruption
- repeated refusal to engage with the actual technical issue

## Scope

This code of conduct applies to repository discussions, issues, pull requests,
and other project communication channels.

## Enforcement

Project maintainers may edit, remove, or moderate content that violates this
code of conduct.
